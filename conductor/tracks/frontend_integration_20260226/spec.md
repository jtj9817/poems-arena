# Track Specification: Phase 6 - Frontend Integration

## Overview
This track focuses on the full frontend integration of the React application (`apps/web`) with the established backend architecture (`apps/api`). The goal is to provide a cohesive, "Digital Letterpress" experience that allows users to engage with a rich corpus of human and AI-generated poems, filterable by topic, and presented with a modern, high-polish "Tinder-like" interaction.

## Functional Requirements
- **Anthology Page (`/anthology`):**
  - Implement a **Sticky Topic Bar** with **Topic Filter Chips** for single-select filtering.
  - On mobile, topic selection should utilize a **Bottom Sheet Filter** for easier reachability.
  - Duel cards must display their **Topic Labels** (e.g., "Nature", "Love").
  - Each duel card should link to the Reading Room for that specific duel.
- **Reading Room Page (`/reading-room`):**
  - Implement a **Tinder-like comparison interaction**:
    - Present two anonymous poems (Exhibit A and Exhibit B).
    - Upon selecting a work, trigger a **"swipe-like" transition animation** to reveal the Verdict.
    - Support fluid navigation to the next duel with a similar transition.
- **Verdict Screen:**
  - Display clear **Source Attribution** (e.g., "Written by Ralph Waldo Emerson", "Generated by Gemini 3 Flash").
  - Show community statistics and win rates for the specific duel.
  - Provide a button to "Enter Next Duel" using the swipe-like transition.
- **Foyer Page (`/`):**
  - Fetch the initial featured duel(s) from `GET /duels` and display the current topic.
  - Provide a seamless transition into the Reading Room.

## Non-Functional Requirements
- **Digital Letterpress Aesthetic:** Maintain the Alabaster/Ink palette and classic serif typography.
- **Performance:** Minimize external package imports; prefer vanilla CSS for animations and transitions.
- **Responsiveness:** Ensure topic bars and bottom sheets are optimized for mobile (iPhone Safari).
- **Type Safety:** Use shared types from `@sanctuary/shared` for all API responses.

## Acceptance Criteria
- [ ] Anthology displays topic chips and cards with labels.
- [ ] Selecting a topic filter updates the duel list via `GET /duels?topic_id=...`.
- [ ] Reading Room features a "swipe-like" transition after a vote is cast.
- [ ] Verdict screen correctly attributes poems to their human or AI sources.
- [ ] Mobile navigation (Bottom Sheet) is functional and accessible.
- [ ] All E2E tests (`packages/e2e`) pass with the new frontend logic.

## Out of Scope
- User accounts and profiles.
- Social sharing integrations.
- Real-time community voting updates (static snapshot on load is sufficient).
