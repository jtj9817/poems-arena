# Track Specification: Phase 6 - Frontend Integration

## Overview
This track focuses on the full frontend integration of the React application (`apps/web`) with the established backend architecture (`apps/api`). The goal is to provide a cohesive, "Digital Letterpress" experience that allows users to engage with a rich corpus of human and AI-generated poems, filterable by topic, and presented with a modern, high-polish interaction flow.

## Backend Prerequisites
- **`GET /api/v1/topics`:** New endpoint returning all canonical topics from the `topics` table.
- **`GET /api/v1/duels?topic_id=...`:** Extend the existing duels listing endpoint to support optional filtering by `topic_id`.
- **Shared Types:** Add `TopicMeta` and `SourceInfo` interfaces to `@sanctuary/shared` to ensure type safety across the stack.

## Functional Requirements
- **Anthology Page (`/anthology`):**
  - Implement a **Sticky Topic Bar** with **Topic Filter Chips** for single-select filtering, dynamically populated from `GET /api/v1/topics`.
  - On mobile, topic selection should utilize a **Bottom Sheet Filter** for easier reachability.
  - Duel cards must display their **`topicMeta.label`** (e.g., "Nature", "Love") instead of raw topic strings.
  - Each duel card should link to the Reading Room for that specific duel.
- **Reading Room Page (`/reading-room`):**
  - Present two anonymous poems (Exhibit A and Exhibit B).
  - Upon selecting a work, display a **Verdict Pop-Up** revealing the result.
  - After the user acknowledges the Verdict, trigger a **"swipe-like" transition animation** to the next duel.
  - Implement **sliding window pre-fetching**: pre-load the next 1–2 duels from the `GET /duels` list order while the user reads the current one, ensuring instant "Next Duel" transitions.
  - When approaching the end of the current page of duel IDs, automatically fetch the next page.
- **Verdict Pop-Up:**
  - Display clear **Source Attribution** (e.g., "Written by Ralph Waldo Emerson", "Generated by Gemini 3 Flash").
  - Show community statistics and win rates for the specific duel.
  - Provide a "Continue" / "Next Duel" button that dismisses the pop-up and triggers the swipe transition.
- **Foyer Page (`/`):**
  - Fetch the initial featured duel(s) from `GET /duels` and display `topicMeta.label`.
  - Provide a seamless transition into the Reading Room.

## Non-Functional Requirements
- **Digital Letterpress Aesthetic:** Maintain the Alabaster/Ink palette and classic serif typography.
- **Performance:** Minimize external package imports; prefer vanilla CSS for animations and transitions.
- **Responsiveness:** Ensure topic bars and bottom sheets are optimized for mobile (iPhone Safari).
- **Type Safety:** Use shared types from `@sanctuary/shared` for all API responses, including `TopicMeta` and `SourceInfo`.

## Acceptance Criteria
- [ ] `GET /api/v1/topics` returns all canonical topics.
- [ ] `GET /api/v1/duels?topic_id=...` filters duels by topic.
- [ ] Anthology displays dynamically fetched topic chips and cards with `topicMeta` labels.
- [ ] Selecting a topic filter updates the duel list via `GET /duels?topic_id=...`.
- [ ] Reading Room displays a Verdict pop-up after a vote is cast.
- [ ] After acknowledging the Verdict, a "swipe-like" transition animates to the next pre-fetched duel.
- [ ] Verdict pop-up correctly attributes poems to their human or AI sources via `sourceInfo`.
- [ ] Mobile navigation (Bottom Sheet) is functional and accessible.
- [ ] Foyer displays `topicMeta.label` for the featured duel.
- [ ] All E2E tests (`packages/e2e`) pass with the new frontend logic.

## Out of Scope
- User accounts and profiles.
- Social sharing integrations.
- Real-time community voting updates (static snapshot on load is sufficient).
